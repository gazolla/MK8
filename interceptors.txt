Security and Access Control

AuthorizationInterceptor — Verifies if the source plugin has permission to invoke a specific capability before routing.
RateLimitInterceptor — Limits invocations per plugin/capability per time window, publishing a capability.error if exceeded.

Observability

TracingInterceptor — Propagates and enriches traceId/spanId in each event, integrating with OpenTelemetry.
MetricsInterceptor — Counts invocations, measures latency, and publishes metrics (e.g., Prometheus, Micrometer).
AuditInterceptor — Persists an immutable log of all events (who invoked what and when).

Resilience

CircuitBreakerInterceptor — Monitors consecutive failures of a capability and temporarily opens the circuit.
RetryInterceptor — Automatically re-sends a capability.invoke in the case of transient capability.errors.
TimeoutInterceptor — Injects a deadline into the event and publishes a capability.error if the result does not arrive in time.

Data Transformation

ValidationInterceptor — Validates the payload against a JSON schema before routing.
TransformInterceptor — Converts payload formats between API versions (contract versioning).
EncryptionInterceptor — Encrypts/decrypts payloads for capabilities marked as sensitive.

Flow and Orchestration

PriorityInterceptor — Reorders the invocation queue based on the priority declared in the payload.
WorkflowInterceptor — Chains capabilities in sequence/parallel based on a declarative definition.
